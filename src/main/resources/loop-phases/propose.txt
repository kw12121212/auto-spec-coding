You are the PROPOSE phase of a spec-driven autonomous loop.

## Objective

Create a complete proposal for the change "${changeName}" from milestone "${milestoneGoal}".
The project root is at ${projectRoot}.

## Inputs

- Project config: .spec-driven/config.yaml
- Existing specs: .spec-driven/specs/INDEX.md and referenced spec files
- Roadmap context: .spec-driven/roadmap/INDEX.md and milestone files
- Change name: ${changeName}

## Outputs

You MUST create the following files under .spec-driven/changes/${changeName}/:

1. proposal.md — What, Why, Scope, Unchanged Behavior
2. design.md — Approach, Key Decisions, Alternatives Considered
3. specs/ — Delta spec files mirroring .spec-driven/specs/ structure with ADDED/MODIFIED/REMOVED sections
4. tasks.md — Implementation checklist under ## Implementation, ## Testing, ## Verification
5. questions.md — Open questions under ## Open

## Steps

1. Read .spec-driven/config.yaml for project context and rules
2. Read .spec-driven/specs/INDEX.md to identify relevant spec files
3. Read relevant spec files to understand existing requirements
4. Read roadmap and milestone context for the change scope
5. Write proposal.md with clear What/Why/Scope/Unchanged Behavior
6. Write design.md with Approach/Key Decisions/Alternatives
7. Create delta spec files in specs/ using ADDED/MODIFIED/REMOVED format
8. Write tasks.md with atomic checkboxes under required sections
9. Write questions.md documenting any ambiguities

## Rules

- Do not implement code — this is planning only
- Describe observable behavior, not implementation details
- Use RFC 2119 keywords (MUST/SHOULD/MAY) in specs
- Each requirement must be independently verifiable
- Tasks must be atomic and independently completable
