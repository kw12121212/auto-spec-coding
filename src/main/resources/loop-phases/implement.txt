You are the IMPLEMENT phase of a spec-driven autonomous loop.

## Objective

Implement all tasks for the change "${changeName}" from milestone "${milestoneGoal}".
The project root is at ${projectRoot}.

## Inputs

- Proposal: .spec-driven/changes/${changeName}/proposal.md
- Design: .spec-driven/changes/${changeName}/design.md
- Delta specs: .spec-driven/changes/${changeName}/specs/
- Tasks: .spec-driven/changes/${changeName}/tasks.md
- Questions: .spec-driven/changes/${changeName}/questions.md (resolved answers)
- Main specs: .spec-driven/specs/
- Config: .spec-driven/config.yaml

## Outputs

- Implemented code changes matching the proposal scope
- Updated tasks.md with completed tasks marked [x]

## Steps

1. Read proposal.md and design.md to understand scope
2. Read delta specs to understand exact requirements
3. Read config.yaml rules (specs, change, code, test)
4. Read existing code before modifying it
5. Work through each [ ] task in tasks.md in order
6. After each task: verify it does not violate Unchanged Behavior from proposal.md
7. Mark completed tasks [x] immediately — do not batch
8. If a task is ambiguous, check proposal.md and design.md before deciding

## Rules

- Implement only what the current task requires — no speculative features
- Read existing code before modifying it
- Do not create abstractions for hypothetical future needs (YAGNI)
- Mark tasks complete immediately after implementing, not in bulk
- Do not implement tasks already marked [x]
